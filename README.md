# Подарок мечты

Веб-сервис для «угадывания» желаемого подарка: опрос по празднику, категории, бюджету и стилю с выдачей рекомендации. Результаты можно отправить в Telegram тому, кто создал ссылку.

## Возможности

- **Два сценария:** «Я хочу получить подарок» (пройти опрос) и «Я хочу узнать, что хочет другой» (получить ссылку и принять результаты в Telegram).
- **Праздники:** 8 марта, годовщина отношений, день рождения, День святого Валентина.
- **Вопросы:** категория подарка, бюджет в рублях, для box — тип наполнения, стиль, пожелания, конкретная идея (сравнение с рекомендацией).
- **Telegram:** пользователь пишет боту `/start`, затем на сайте вводит свой логин и получает персональную ссылку. После прохождения опроса по ссылке результаты приходят в чат с ботом.
- **Ссылки действуют 30 дней.**

## Запуск

1. Установить зависимости:
   ```bash
   npm run install:all
   ```
2. Создать файл `.env` в корне (или использовать уже созданный) и указать:
   ```
   TELEGRAM_BOT_TOKEN=ваш_токен_бота
   PORT=3000
   ```
3. Собрать фронтенд:
   ```bash
   npm run build
   ```
4. Запустить сервер:
   ```bash
   npm start
   ```
   Сайт: http://localhost:3000

Для разработки фронтенда с горячей перезагрузкой:
- в одном терминале: `npm start` (бэкенд);
- в другом: `cd frontend && npm run dev` (фронтенд на Vite, прокси к API).

## Дополнительная логика и развитие сервиса

- **Ограничение срока ссылки** — ссылка на опрос действительна 30 дней, затем считается устаревшей.
- **Один опрос — один получатель:** каждая ссылка привязана к одному Telegram-чату, результаты приходят только ему.
- Идеи для монетизации: премиум-рекомендации с партнёрами (магазины, бренды), платная расшифровка «что именно купить» с ссылками, подписка на напоминания перед праздниками.
- Возможные улучшения: напоминание в боте за N дней до праздника, сохранение истории опросов в боте, интеграция с партнёрами по категориям (подарочные боксы, впечатления).

## Безопасность

- Токен бота храните только в `.env`; файл добавлен в `.gitignore`.
- Не публикуйте `.env` и токен в репозитории.
